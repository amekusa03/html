# ESP32 Air Conditioner Control (+ SwitchBot Web API)

2026/07/15 / Arduino / ESP32 C3

暑くてエアコンを入れるて数時間すると寒い。そうこうするうちにまた暑くなる。これを繰り返していると不経済
そこでESP32とSwitchbotを使って快適な温度を保てるようにする。

## 1. プロジェクト概要

本プロジェクトは、追加のサーバー機器を購入せず、USB電源のみで動作するマイクロコントローラー「ESP32-C3」と高精度温湿度センサー「SHT30」を用いて、室温・湿度を24時間監視し、SwitchBotハブミニ経由でエアコンを自動制御することで、「冷えすぎ（寒さ）」や「エアコンが効かない（暑さ）」といった室内の温度ムラを緩和し、快適な室内環境を維持するシステムである。

## 2. システム構成・役割分担

本システムはクラウド（SwitchBot Web API）を経由したローカル・スタンドアロン制御を行う。 

```text
`[室内環境] ➔ (I2C) ➔ [ESP32-C3 ＋ SHT30] (親機なしで単体動作)
       │ (Wi-Fi / HTTPS)
       ▼
[SwitchBot Web API]
       │ (クラウド経由)
       ▼
[SwitchBot ハブミニ] ➔ (赤外線) ➔ [エアコン] `
```

### 2.1 構成部品と役割

- **子機兼制御本体: ESP32-C3 開発ボード **（電源：5V USB給電のみ） 
  - SHT30から10〜30秒間隔で室温を取得。 
  - 制御ロジック（ヒステリシス）の判定および管理。 
  - Wi-Fi経由でSwitchBot Web APIへHTTPSリクエスト（制御コマンド）を直接送信。 

- **センサー: SHT30 **（温度・湿度センサーモジュール） 
  - 生活動線上の高さ（床上70〜100cm）に設置し、正確な体感温度を計測。 
  - ESP32-C3の通信発熱による影響を受けないよう、物理的に数センチ離して配置。 

- **赤外線中継機: SwitchBotハブミニ **（既存導入済み） 
  - クラウドからの要求を受け取り、エアコンへ赤外線信号を送信。 

- **操作対象: **既存のエアコン 
- **外部サービス: SwitchBot Web API (v1.1) **
  - デバイス一覧の取得、およびエアコンの遠隔操作（ `setAll `コマンド）に使用。

## 3. ハードウェア配線（I2C接続）

ESP32-C3とSHT30は以下のピンマップで接続する。 

SHT30 側 ESP32-C3 側 信号名・役割 備考 **VCC ****3.3V **電源供給 **GND ****GND **グランド **SDA ****GPIO 8 **I2C データ線 プログラム内で任意に変更可能 **SCL ****GPIO 9 **I2C クロック線 プログラム内で任意に変更可能 ※注意1: Wi-Fi接続時の突入電流（電流スパイク）による不安定化を防ぐため、品質の良いUSBケーブルを使用すること。 ※注意2: GPIO 9はESP32-C3のブートストラップピンです。接続するSHT30モジュールに強力なプルダウン抵抗がある場合、起動不良（書き込みモードへの遷移）を起こす可能性があります。その場合は、I2Cピンを別のGPIO（例: SDA=GPIO 5, SCL=GPIO 6など）へ変更してください。

## 4. 制御ロジック（エアコン不快感緩和仕様）

チャタリング（エアコンの頻繁なオンオフ）およびハンチング現象を防ぐため、 **1.5℃〜2.0℃の幅（ヒステリシス） **を持たせた2状態フラグ管理を行う。 


#### 4.1 自動制御ルール


- **【冷えすぎ（肌寒さ）防止トリガー】 **
  - **条件 **: 室温が **24.5℃以下 **に低下、かつ 現在「通常モード（または判定前）」の場合。 
  - **アクション **: エアコンの設定を **「冷房 27℃ / 風量：しずかに（微風） / 風向：上向き」 **へ変更し、状態を「弱風モード」へ移行する。 
  - **目的 **: 部屋を完全に冷やし切らず、微風と上向き送風で冷気の直撃を緩和する。 

- **【効かない（暑さ）対策トリガー】 **
  - **条件 **: 室温が **26.5℃以上 **に上昇、かつ 現在「弱風モード」の場合。 
  - **アクション **: エアコンの設定を **「冷房 25℃ / 風量：自動」 **へ変更し、状態を「通常モード」へ移行する。 
  - **目的 **: 部屋が本格的に暑くなる前に先手を打って冷やし始める。 


#### 4.2 信頼性・安全対策仕様


- **【ガードタイム（最低送信間隔の制限）】 **
  - エアコンの頻繁な設定変更（チャタリング）によるコンプレッサー負荷軽減およびSwitchBot APIの制限（1日10,000回まで）回避のため、一度コマンドを送信した後は **最低15分間、次のコマンド送信を行わない **ようガードタイムを設ける。 

- **【状態（ステータス）の永続化】 **
  - ESP32-C3が電源瞬断やWi-Fi切断等で再起動した場合に状態を見失わないよう、現在の制御状態（通常モード / 弱風モード）を内蔵Flashメモリ（ `Preferences `ライブラリ）へ随時書き込み保存する。 
  - 起動時に保存された状態がない場合は、安全のため「室温が25.5℃未満なら弱風モード、25.5℃以上なら通常モード」として初期化する。 

- **【センサー異常時のフェールセーフ】 **
  - SHT30とのI2C通信エラーや測定値の異常（ `NAN `など）を検出した場合は、エアコンの暴走を防ぐため制御処理を一時中断し、シリアルモニター等へエラーを出力して待機する。

## 5. ソフトウェア仕様（通信および認証）

SwitchBot Web API v1.1 の仕様に準拠したセキュアな通信をESP32-C3内で実装する。 


#### 5.1 認証情報の準備

事前にスマホアプリの「開発者オプション」から以下の情報を取得し、プログラムへ埋め込む。 


- **Token (トークン) **
- **Client Secret (クライアントシークレット) **
- **エアコンのデバイスID **（事前に一覧取得用コードを用いて特定する） 

#### 5.2 APIリクエスト仕様


- **エンドポイント **: `https://api.switch-bot.com/v1.1/devices/{device_id}/commands `
- **リクエストメソッド **: `POST `
- **必須HTTPヘッダー **: 
  - `Content-Type: application/json `
  - `Authorization: {Token} `
  - `sign: {HMAC-SHA256署名文字列} `（Secret、ミリ秒タイムスタンプ、Nonceの結合文字をHMAC-SHA256ハッシュ化し、Base64エンコード後に **すべて大文字（Uppercase） **へ変換したもの） 
  - `t: {ミリ秒タイムスタンプ} `
  - `nonce: {ランダムな文字列/UUID} `

- **送信ペイロード（JSON例） **: 
```
`{
  "command": "setAll",
  "parameter": "27,cool,1,on", 
  "commandType": "command"
} `
```
*(※parameter書式: "設定温度,運転モード,風量,電源状態")*

## 6. 開発ロードマップ（今後のステップ）

- **Step 1 **: SHT30の値を正しくシリアルモニターに表示させる（配線確認）。 
- **Step 2 **: 1度だけ実行するコードを書き、SwitchBotに登録されている機器の「エアコンのデバイスID」をログから特定する。 
- **Step 3 **: ESP32-C3内で、API v1.1必須の「ミリ秒タイムスタンプ取得（NTPサーバー同期と同期完了待ち）」と「HMAC-SHA256署名（シグネチャ）の自動生成（大文字変換を含む）」を実装し、単体テストする。 
- **Step 4 **: `Preferences `による制御状態の永続化と、ガードタイム処理を実装する。 
- **Step 5 **: 制御ロジックおよびフェールセーフ（センサー異常検出）を組み込み、本運用を開始する。
