Google Nest Hub Home Automation Scripting

Google Nest Script

Google Nest Hubの高度なカスタマイズを実現するためのスクリプト

About this Project

Google Nest HubのGUIでは設定できない高度なカスタマイズを実現するためのスクリプトを紹介します。 GUIでは実現できない細かい設定や、特定の条件下での動作を実現するためのスクリプトを作成し、 Google Nest Hubの機能を最大限に引き出すことを目指しています。

Implementation

今回はGUIでは実現できない「第2木曜日」とのトリガを実装するスクリプトを紹介します。 Google Nest Hubのスクリプト設定では、特定の条件に基づいて動作をトリガーすることができますが、 GUIでは「第2木曜日」といった複雑な条件を設定することはできません。 そこで以下のようなスクリプトを作成し、Google Nest Hubのスクリプト設定に組み込むことで、 「第2木曜日」に特定のアクションを実行することが可能になります。 

metadata:
  name: びん・缶の日リマインダー
  description: 第2木曜日の朝にNest Hubでお知らせする

automations:
  - starters:
      - type: time.schedule
        at: 07:00
        weekdays:
          - THURSDAY

    condition:
      type: time.between
      after: 7
      before: 15

    actions:
      - type: assistant.command.Broadcast
        # 修正箇所: home.command ではなく assistant.command を使用
        message: "今日は、びん・缶のごみ出しの日です。"
解説
  • metadata: スクリプトの基本情報を定義します。nameはスクリプトの名前、descriptionはスクリプトの説明です。
  • automations: 自動化のルールを定義するセクションです。
  • starters: 自動化を開始する条件を定義します。ここでは、毎週木曜日の7:00にトリガーされるように設定しています。
  • condition: アクションが実行される前に満たすべき条件を定義します。ここでは、日にちが7日から15日であることを条件としています。これにより第２木曜日を特定できます。
  • actions: 条件が満たされたときに実行されるアクションを定義します。ここでは、Google Nest Hubで特定のメッセージを放送するアクションを設定しています。

ポイント

メッセージはhome.commandではなく、assistant.commandを使用してbroadcastする必要がありました。 home.commandはGoogle Homeデバイス全体に対してコマンドを送信するためのものであり、 assistant.commandはGoogle Nest Hubなどの特定のデバイスに対してコマンドを送信するためのものです。 今回のようにGoogle Nest Hubでメッセージをお知らせする場合は、assistant.commandを使用する必要があります。

Key Features

  • 柔軟なスケジュール設定: GUIの制限を超えた、特定の日付や複雑な条件での自動化が可能。
  • デバイスブロードキャスト連携: Nest Hubの機能を直接制御し、音声によるリマインドを実現。
