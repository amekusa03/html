ソフトPLCの定義（前提）

  • ソフトPLCまたはソフトウェアPLCとは、 制御ロジックソフトウェアの一部または全部がPLCハードウェアから切り離されており、 ソフトウェアが制御ロジックワークロードの実行を異なるハードウェア上で行うことを可能にする 制御システムのことです。 また、仮想化PLCとは、仮想化されたハイパーバイザーの上のコンピューティング環境に配置されたソフトPLCのことです。

対して ハードウェアPLC

三菱電機のQシリーズ、横河電機のFA-M3やオムロンのNJは、 ハードウェア的にもソフトウェア的にも独自仕様となっている。 これらをソフトウェアPLCに対し、ハードウェアPLCという。

CODESYSとは

  • CODESYSは国際標準規格 IEC 61131-3に準拠した、 Software PLC / -Motion / -HMIです。
  • PLC, Motion, HMIの全てをシームレスに開発可能なCODESYS統合開発環境、 いかなるプラットフォームにもアダプテーション可能なCODESYSランタイムにより、 全世界に400社以上のOEM顧客、数千社のエンドユーザーでの稼働実績を有します。

IEC 61131-3とは

  • IEC 61131-3とは、国際電気標準会議（IEC）が発行した PLC用標準規格 IEC 61131シリーズの第3部（全10部）である。 1993年12月に初版が発行され、 現行版は2013年2月に改訂された第3版である。 また、それを元にJISでもJIS B 3501が制定されている。
  • 第3部はプログラミング言語仕様を定義したものであり、 PLCの基本的なソフトウェア・アーキテクチャとそこで利用できる以下の5種類のプログラム言語を定義したものである。

定義されたプログラム言語

  • ラダー・ダイアグラム（LD言語）
  • ファンクション・ブロック・ダイアグラム（FBD言語）
  • ストラクチャード・テキスト（ST言語）
  • インストラクション・リスト（IL言語）

CODESYS統合開発環境

オープン化を実現するオールインワン開発環境

PLCプログラム、Motion制御、HMI作成、フィールドバス設定といった様々な プログラミングや設定をオールインワンで行うことが可能です。 従来のようにPLC、Motionコントローラー、HMIといった各デバイスに それぞれ提供されるツールを習熟する必要がありません。

オールインワンでシームレスに開発できることで開発効率が向上します。

CODESYSを利用することで国際標準規格に 準拠したプログラミングが可能となり、 オープン化も同時に実現することが可能となり、 ソフトウェア資産の再利用性も高まります。

このCODESYS開発環境は全世界のCODESYSユーザーが無償で利用可能です。

まとめ、注意点など

  • V3.5 SP15からV3.5 SP18へUpdateすると、ラダーにビルドエラーが発生。 SPが異なるとプロジェクトに互換性がない事が発生した。
  • 使用にはユーザー登録(メールアドレス登録)が必要。
  • Code Meterが標準インストールされる ライセンス認証ソフトウェア、 アンインストールし、プログラミングとシミュレーションが動作する事が確認した。
  • 無料でラダー等、プログラミングからシミュレーション可能 すぐに業務の開発環境が変わる事はないと思うが、 ラダー学習用にも。
